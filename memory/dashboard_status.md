## Design And Interaction Notes

The match hover and fixed modal have two pages:

- stats page by default
- models page after navigating with the right arrow

The stats page shows:

- xG scatter by model
- scoreline consensus heatmap
- 1X2 probability bars/ranges
- recent friendlies and preparation goal reference

The models page shows individual model predictions, expected goals, 1X2 probabilities and EV/pick information.

Current stats details:

- xG scatter includes a star for weighted recent-friendly goal reference.
- Heatmap cells show percentage as the main value and the exact model count as a small number. Do not append `mod.`.
- 1X2 chart has two lanes: top lane is model consensus average; bottom lane is the current live preferred/best-performing model. Gold whisker remains min/max model range.
- Recent friendlies show the last 5 friendlies per team and weighted goal reference.

## Real Results

When a match has a finished/completed status and recorded scores in the local database, the dashboard can show the real result and use it in live model ranking.

Before real 2026 results are available, the best-current model fallback is `weighted_points_ensemble`/default logic. After real results exist, pending match picks use the live 2026 best-performing model from frozen pre-match predictions. Completed matches keep their frozen original pick.

## Validation 2026 Live

The Validation section has live current-tournament panels:

- `2026 · Max Pts.`
- `2026 · Más Probable`

Rows in those panels are clickable. Click opens `val-hover` with a compact model-vs-real comparison for completed matches only.

If opened from `Max Pts.`, the hover evaluates `score` first.
If opened from `Más Probable`, the hover evaluates `top` first.

The hover includes:

- selected-mode points
- exacts and hits
- 1X2 hit rate
- average probability assigned to the real 1X2 outcome
- average xG error
- compact match-by-match comparison

Do not edit generated HTML in `outputs/` directly.
